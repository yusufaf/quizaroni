# Quizaroni

[![Netlify Status](https://api.netlify.com/api/v1/badges/4145a48b-b7d8-4a04-a158-b922b810e64e/deploy-status)](https://app.netlify.com/projects/quizaroni/deploys)

A flashcard and quiz study application. Create study sets, organize cards with labels and notes, study with multiple modes, and share publicly.

Live: [quizaroni.netlify.app](https://quizaroni.netlify.app)

## Tech Stack

- **React 18** + **TypeScript** on **Vite+** (`vp`) ‚Äî unified toolchain wrapping Vite, Vitest, Oxlint, Oxfmt
- **Material UI v5**, Emotion, Framer Motion
- **State**: Redux Toolkit (RTK Query), Zustand, TanStack Query
- **Routing**: React Router v6
- **Auth**: AWS Amplify v6 + Cognito
- **i18n**: react-i18next (English, Spanish)
- **Storage**: Dexie (IndexedDB) for offline cache
- **Deploy**: Netlify (frontend), AWS API Gateway + Lambda (backend, separate repo)

## Prerequisites

- Node.js 24.15.0 (pinned in `.node-version`)
- pnpm 10.x (via `packageManager` field ‚Äî Vite+ wraps it)

## Getting Started

```bash
pnpm install
vp dev          # starts dev server on http://localhost:3000
```

Other commands:

```bash
vp build        # production build ‚Üí dist/
vp preview      # preview production build
vp check        # format + lint + typecheck
vp test         # run Vitest
```

`vp` is the Vite+ CLI. If it's not on your path, install it from [vite.dev/plus](https://vite.dev/plus/).

## API

Dev requests to `/api/*` are proxied to AWS API Gateway. Production uses Netlify redirects defined in `netlify.toml`.

The backend (AWS Lambda + DynamoDB + S3, deployed via CDK) lives in a separate repository.

## Authentication

The app is wrapped by Amplify's `Authenticator` in `src/index.tsx`. Cognito user pool config is in `src/aws-exports.js` (auto-generated by `amplify pull` ‚Äî values shipped to the client are non-secret identifiers). Route protection lives in `src/AppRoutes.tsx`.

To use a different Cognito pool, run `amplify pull` against your own Amplify project, or replace the values in `aws-exports.js` directly.

## Internationalization

User-facing strings live in `src/i18n/locales/{en,es}/common.json`. Use the `useTranslation` hook:

```tsx
const { t } = useTranslation();
<Typography>{t('home.title')}</Typography>
```

Add new keys to both language files.
